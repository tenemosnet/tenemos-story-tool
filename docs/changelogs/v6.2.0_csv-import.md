## 概要

チャットブースト（LINE配信管理ツール）の配信履歴CSVを、カレンダー画面にドラッグ&ドロップで取り込む機能を実装。tenemos-brain（月次施策検証サマリー）でLINE配信の実績データが不足している問題への対応。

## DB変更

- `finished_contents` に `chatboost_delivery_id TEXT` カラム追加（部分インデックス付き）
- `chatboost_delivery_stats` テーブル新規作成（配信統計ログ：送信数・開封率・クリック率等）

## 処理フロー

1. カレンダー画面でCSVをドラッグ&ドロップ（またはクリックでファイル選択）
2. Shift-JIS→UTF-8変換 → RFC4180パース（改行含むフィールド対応）
3. 2026/6/1以降の配信を対象に、同日のLINE配信予定を検索
4. 既存レコードあり → タイトル・is_done・chatboost_delivery_idのみ更新（line_delivery_type保持）
5. 既存なし → 新規レコード作成（body空、type=line、is_done=true）
6. 配信統計を `chatboost_delivery_stats` にUPSERT
7. 結果モーダルで更新/新規/スキップ/エラー件数を表示

## 設計上の重要判断

| 判断 | 選択 | 理由 |
|------|------|------|
| 文字コード変換 | `TextDecoder('shift_jis')` | Cloudflare Workers互換（iconv-liteはNode.js Buffer依存） |
| CSVパーサー | 新規実装 | mail.tsはfs依存・UTF-8固定・3列構造で流用不可 |
| UPDATE範囲 | title, is_done, chatboost_delivery_idのみ | line_delivery_type保持で既存のbroadcast/segment設定を壊さない |
| 重複防止 | chatboost_delivery_id | CSV再取込時は全件スキップ |
| body デフォルト | 空文字 `''` | NOT NULL制約対応、既存POSTと同パターン |
| 統計データ | 別テーブル `chatboost_delivery_stats` | finished_contentsの責務と分離、UNIQUE制約でUPSERT可 |
| ファイル選択 | D&D + クリックの2系統 | モバイル・アクセシビリティ対応 |
| 結果表示 | モーダル | 構造化された複数情報を表示するためトーストより適切 |
